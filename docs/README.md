# OctoAcme プロジェクト管理ドキュメント

このフォルダーでは、OctoAcme が採用しているプロジェクト管理プロセスのドキュメントを管理しています。  
プロジェクトの立ち上げから完了・振り返りまでを一貫して実行するためのガイドラインをまとめています。

---

## OctoAcme のプロジェクト管理プロセスの概要

OctoAcme のプロジェクト管理は、以下の原則に基づいて運営されています。

- **顧客価値・使いやすさ重視**: 顧客に価値をもたらすことを最優先とします。
- **反復的デリバリー**: 小さな単位でテスト可能な成果物を継続的にリリースします。
- **明確なオーナーシップ**: 各プロジェクトには担当 PM（Project Manager）とプロダクトリードを設置します。
- **データに基づく意思決定**: 計測と根拠を重視し、証拠をもとに改善を繰り返します。
- **標準アーティファクトの活用**: プロジェクト憲章、リスク管理台帳、スプリントバックログなど共通成果物を利用します。

### プロジェクトライフサイクル（概要）

1. **立ち上げ（Initiation）** – 課題定義、ステークホルダー特定、大まかなスケジュール策定
2. **計画（Planning）** – スコープ、リソース、マイルストーン、依存関係の整理
3. **実行（Execution）** – 構築・テスト・レビュー・反復
4. **リリース（Release）** – デプロイ・検証・告知
5. **振り返り（Retrospective）** – 学びの記録と次のステップへのフィードバック
